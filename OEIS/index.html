March 2025

OEIS is a database of interesting numeric sequences that first launched in book format in 1973 and later moved online in 1996.

Here is what it looked like back then.

Compare that to today.

Some things changed. But other things stayed the same. The homepage still uses system fonts, and the HTML is just as lightweight at 3.8KB gzipped, which is one way to make the webpage load extremely fast.

But it's also 2025 and the UI could use an update. What follows are a few simple suggestions.

Banner

The main issue with the current banner is the kerning. The letters in the logo should not overlap (unless you want to emulate Kia), and the letters in the title should be closer together. Also, “ON-LINE” is now overwhelmingly dash-free. The logo should be bigger and vertically centered with respect to both title and subtitle combined.

Here's one fix to these problems:

The logo text and banner text are now set in the same typeface to achieve visual consistency. I chose Ruwudu to preserve the flourish on the letter Q, and for its taller, more authoritative letterforms, not unlike some older textbooks. Regrettably, the numerals in Ruwudu do not descend like in the original banner.

On the logo itself, the red numbers are desaturated to put emphasis on the OEIS lettermark. The logo is also now more vertically “balanced”. Here are the lines that helped me center the logo.

Surely a professional can adjust it better.

As an added bonus, the new banner is an SVG file, so after subsetting Ruwudu, the new banner file size is ~7KB, which is significantly smaller in file size than the original 42KB PNG.

The above suggestions are easily adoptable since we're merely sprinkling in CSS or replacing PNG with SVG (plus a font file).

On the other hand, changing how the information within each table row is presented (like converting the formulas into MathJax, or offering code syntax highlighting) might require client-side RegEx parsing with exceptional cases, or going through the database itself and reformatting the information in each entry.
