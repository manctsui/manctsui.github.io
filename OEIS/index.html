UI/UX OEIS

OEIS is a database of interesting numeric sequences that first launched in book format in 1973 and later moved online in 1996.

Here is what it looked like back then.

Compare that to today.

Some things changed, but some things stay the same. Let me describe a few simple changes to update OEIS's UI.

The banner

The main issue with the current banner is the kerning. The letters in the logo should not overlap (unless you are trying to emulate Kia), and the letters in title should be closer together. By the 2010s, “ON-LINE” is dash-free. The logo should be bigger and vertically centered with respect to the title and subtitle combined.

Here's one fix to these problems:

Notice that the banner text is now set in the same type to achieve visual consistency. I chose Ruwudu to preserve the flourish on the letter Q, and for its authoritative, taller letterforms, not unlike some older textbooks. Regrettably, the numerals in Ruwudu do not descend like in the original banner.

On the logo itself, the red numbers are desaturated to put emphasis on the OEIS letters. The logo feels more vertically “balanced”, though I'm sure a professional can tweak it more. Here are the lines that helped me center the logo.

Overall, the new banner is an SVG, so after subsetting Ruwudu, the new banner file size is ~7KB, compared to the original 42KB PNG.

The above suggestions are easily adoptable since we're merely sprinkling in CSS or replacing PNG with SVG (plus a font file).

On the other hand, changing how the information within each table row is presented (like converting the formulas into MathJax, or offering syntax highlighting for Maple, Mathematica, etc. code) might require some JS + RegEx on the client-side with many exceptional cases, or going through the database itself and reformatting the information in each entry.
